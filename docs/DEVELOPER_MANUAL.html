1. Overview

Wealth OS is a full-stack personal wealth management system designed for Saudi and international investors. It tracks brokerage holdings and private deal investments, provides portfolio analytics, supports CSV/XLSX imports from 11 Saudi platforms, and delivers a modern dashboard with allocation charts and risk insights.

Migration Commands

docker compose exec backend alembic upgrade head

# Create a new migration after model changes
docker compose exec backend alembic revision --autogenerate -m "your description"

# Roll back one step
docker compose exec backend alembic downgrade -1

docker compose exec backend python -m app.scripts.seed

Seed creates: 11 Saudi platforms, 1 demo user, 8 sample deals, brokerage transactions, 5-year mock price history.

7. Backend Architecture

Request Flow

HTTP Request
  → FastAPI router (api/v1/*.py)
  → Pydantic schema validation
  → Business logic / calculator
  → SQLAlchemy ORM
  → PostgreSQL
  → Pydantic response schema
  → JSON response

Authentication

JWT bearer tokens. Every protected endpoint uses the get_current_user dependency from api/deps.py.

Token flow:

  1. Call POST /api/v1/auth/login — receives access_token
  2. Include in all subsequent requests: Authorization: Bearer <token>

13. Adding a New Parser

  1. Open backend/app/parsers/generic_parsers.py
  2. Add a new class extending BrokerageParser or DealParser
  3. Implement the parse(file_path) method — return a list of transaction dicts
  4. Register the parser in backend/app/api/v1/imports.py
  5. Add a CSV template to templates/ so users know the expected column format

14. Adding a New API Endpoint

  1. Create or open a router file in backend/app/api/v1/
  2. Add your route using @router.get() or @router.post()
  3. Use Depends(get_db) and Depends(get_current_user) for database and auth access
  4. Register the router in backend/app/api/v1/router.py
  5. Add a Pydantic schema in backend/app/schemas/ for request/response validation

15. Deployment

GCP (Cloud Run)

See infra/gcp/notes.md for full steps.

  1. Push images to Google Artifact Registry
  2. Deploy backend to Cloud Run
  3. Deploy frontend to Cloud Run
  4. Use Cloud SQL for PostgreSQL
  5. Set environment variables via Secret Manager

Azure (App Service)

See infra/azure/notes.md for full steps.

  1. Push images to Azure Container Registry
  2. Deploy via Azure App Service (containers)
  3. Use Azure Database for PostgreSQL
  4. Configure environment variables in App Service settings

Production Checklist

  • Change SECRET_KEY to a random 32+ character string
  • Set ENVIRONMENT=production
  • Set a strong POSTGRES_PASSWORD
  • Set CORS_ORIGINS to your actual frontend domain
  • Enable HTTPS (handled automatically by Cloud Run / App Service)
  • Set up automated database backups
Security Note: Never commit your .env file to version control. It is listed in .gitignore by default.

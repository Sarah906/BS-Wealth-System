1. Introduction

Wealth OS is a personal wealth management system that helps you track all your investments in one place. Whether you invest through Saudi brokerage platforms (Derayah, Al Rajhi, Alinma) or in private deals and funds, Wealth OS gives you a complete picture of your portfolio performance.

What You Can Do

  • Track brokerage holdings — stocks, ETFs, sukuk across all your accounts
  • Monitor private deals — capital calls, distributions, XIRR returns
  • Import statements — upload CSV or Excel files from your broker
  • View analytics — allocation charts, cashflow timelines, portfolio summary
  • Get insights — automatic risk alerts and recommendations

5. Insights Page

Wealth OS automatically analyzes your portfolio and flags potential issues. Each alert has a severity level:

High — Requires immediate attention
Medium — Worth reviewing soon
Low — Informational, no urgent action needed
Info — General portfolio observation

Types of Alerts

Alert TypeWhat It Means
Concentration RiskToo much of your portfolio in one stock or sector
Currency ExposureHigh exposure to a non-SAR currency without hedging
Negative ReturnA position is significantly down from your cost basis
Stale Price DataPrice data has not been updated recently
Low DiversificationPortfolio spread across too few assets

Each alert includes a Recommendation suggesting what action to consider.

6. Imports Page

Use this page to upload your brokerage statements and import transaction history.

How to Import a File

1
Download your statement from your broker's platform as a CSV or Excel file
2
Go to the Imports page in Wealth OS
3
Click Upload File and select your CSV or XLSX file
4
Select your Platform from the dropdown (e.g. Derayah, Al Rajhi)
5
Click Preview — the system shows you a sample of the rows it will import
6
Review the preview, then click Run Import to save the data
7
The system shows rows imported, rows skipped, and any errors

Import Status Meanings

StatusMeaning
PendingFile uploaded, not yet processed
PreviewFile parsed and ready to review
ProcessingImport is currently running
CompletedAll rows imported successfully
PartialSome rows imported, some had errors
FailedImport could not be completed — check error log

Import History

The bottom of the Imports page shows all previous imports with their status, date, and row counts. You can review any past import to see what was processed.

Duplicate Prevention: The system checks for duplicate transactions using unique identifiers. Importing the same file twice will not create duplicate records.

9. Frequently Asked Questions

Why is my portfolio value showing as zero?

You need to import transaction data first. Go to the Imports page, upload a file from your broker, and run the import. After that, the Overview page will show your holdings.

Why does my return percentage look different from my broker's app?

Wealth OS uses the FIFO method (First In, First Out) to calculate cost basis, which may differ from the method your broker uses. Also, Wealth OS includes fees in the cost basis calculation.

Can I track investments in USD or other currencies?

Yes. Wealth OS supports multiple currencies. Each transaction can have its own currency, and the system converts values to SAR using FX rates for the portfolio summary.

What happens if I import the same file twice?

The system uses unique identifiers to detect duplicates. If a transaction already exists, it will be skipped and counted in the "rows skipped" count.

My broker is not in the supported list. Can I still import?

Yes — use the standard brokerage or deal CSV template from the templates/ folder. Fill in your transactions manually in that format and import it using the "Generic" parser option.

How do I log out?

Currently, your session expires automatically after 24 hours. To log out immediately, clear your browser's local storage or close the browser tab.

Where is my data stored?

All data is stored locally in a PostgreSQL database running on your machine inside Docker. Nothing is sent to any external server.

Need help? Contact your system administrator or the development team for support.
